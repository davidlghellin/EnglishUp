# Tech & Big Data Vocabulary

## Professional Technical Terms Made Simple

---

## Common Tech Terms

### Data Pipeline

**What it means**: The steps data moves through from collection to analysis.

```
collection → cleaning → processing → storing → analyzing
```

**Example sentences:**
- "This part of the **pipeline** is failing."
- "We need to fix the data **pipeline**."
- "The **pipeline** runs every night at 2am."

---

### Compute Overhead

**What it means**: Extra work the system must do (extra processing/resources).

**Example sentences:**
- "Spark helps reduce **compute overhead** when memory is limited."
- "This process has too much **overhead**."
- "We need to minimize the **overhead** of this operation."

---

### Resource Allocation

**What it means**: How CPU, RAM, and tools are distributed among tasks.

**Example sentences:**
- "We need better **resource allocation** for large jobs."
- "The **resource allocation** is not balanced."
- "Who handles **resource allocation** in your team?"

---

### Workload Management

**What it means**: Organizing tasks so work is manageable.

**Example sentences:**
- "My **workload** this week has been insane — 80 hours."
- "We need better **workload management**."
- "The team is struggling with **workload** distribution."

---

### Edge Case

**What it means**: A rare situation that causes unexpected behavior.

**Example sentences:**
- "This bug only appears in an **edge case**."
- "Did you test for **edge cases**?"
- "We found several **edge cases** we hadn't considered."

---

### Scalability Issues

**What it means**: Problems that appear when a system grows larger.

**Example sentences:**
- "The framework starts having **scalability issues** at high volume."
- "We're seeing **scalability** problems with 1M+ users."
- "The system doesn't **scale** well."

## More Tech Vocabulary

| Term | Simple Definition | Example |
|------|-------------------|---------|
| **deploy** | put code/system live | "We'll **deploy** the fix tomorrow." |
| **debug** | find and fix errors | "I spent 3 hours **debugging** this." |
| **refactor** | rewrite code to be cleaner | "We need to **refactor** this module." |
| **rollback** | go back to previous version | "Let's **rollback** to the last stable version." |
| **downtime** | time when system is not working | "We had 2 hours of **downtime** yesterday." |
| **latency** | delay/response time | "The **latency** is too high." |
| **throughput** | amount of data processed | "We need to increase **throughput**." |
| **bottleneck** | point that slows everything | "The database is the **bottleneck**." |
| **sprint** | work period (usually 2 weeks) | "We'll finish it in the next **sprint**." |
| **standup** | short daily meeting | "Let's discuss this at the **standup**." |
| **backlog** | list of tasks to do | "Add it to the **backlog**." |

## Explaining Tech to Non-Technical People

### Use Analogies

| Technical | Simple analogy |
|-----------|----------------|
| Data pipeline | "**It's like** a factory assembly line for data." |
| Cloud computing | "**Think of it like** renting a computer that lives on the internet." |
| Database | "**It's similar to** a digital filing cabinet." |
| API | "**It's like** a waiter - takes your order and brings back what you asked for." |
| Cache | "**Think of it like** a short-term memory - quick to access but temporary." |
| Bug | "An error in the code, **like** a typo that breaks things." |
| Server | "A powerful computer that serves information to other computers." |

### Useful Phrases

- "**Basically**, what this does is..."
- "**In simple terms**..."
- "**Think of it like**..."
- "**It's similar to**..."
- "**Imagine**..."

### Example

❌ Technical jargon:
"We're experiencing latency issues in the ETL pipeline affecting downstream analytics."

✅ Simple explanation:
"**Basically**, there's a delay in how we process data. **Think of it like** a traffic jam - the data is stuck and not getting to where it needs to go fast enough."

## Meeting Phrases for Tech

### Asking for Clarification

- "Could you **explain that** in simpler terms?"
- "What do you mean by [technical term]?"
- "Could you give me an **example**?"
- "Just to clarify, are you saying...?"

### Asking to Slow Down

- "Could you **speak a little slower**, please?"
- "Could you **repeat that**, please?"
- "Sorry, I didn't catch that."
- "Let me make sure I understand..."

### Giving Updates

- "The project is **on track**." (going well)
- "We're **behind schedule**." (late)
- "We hit a **blocker**." (something stopping progress)
- "We need to **prioritize** this."
- "This is a **high priority** / **low priority** issue."

### Discussing Problems

- "We're seeing an **issue** with..."
- "There's a **bug** in the system."
- "This is causing **problems** because..."
- "The **root cause** is..."

## Grammar for Tech Conversations

### "There is" vs "There are"

| Singular | Plural |
|----------|--------|
| There **is** a bug | There **are** bugs |
| There **is** an issue | There **are** issues |
| There **is** one problem | There **are** many problems |

❌ "There is people waiting"
✅ "There **are** people waiting"

### Using "AGO" for Past Time

| Wrong ❌ | Correct ✅ |
|----------|-----------|
| two Mondays **before** | two Mondays **ago** |
| three weeks **before** | three weeks **ago** |
| yesterday **before** | just use "yesterday" |

**Examples:**
- The meeting was **two weeks ago**.
- I deployed the fix **three days ago**.
- We started this project **six months ago**.

### Articles with Meetings/Events

| Wrong ❌ | Correct ✅ |
|----------|-----------|
| I went to meeting | I went to **the** meeting |
| We had standup | We had **a** standup |
| I'm in call | I'm **on a** call |

## Practice Exercises

### Exercise 1: Explain Simply

Explain these to a non-technical person:

1. "The server is down"
2. "We need to debug the code"
3. "There's a bottleneck in the system"

<details>
<summary>Possible Answers</summary>

1. "The computer that runs our website has stopped working."
2. "We need to find and fix the errors in the program."
3. "There's one part of the system that's slowing everything else down - like one slow car causing traffic."
</details>

### Exercise 2: Correct the Grammar

1. There is many users online.
2. I went to meeting yesterday.
3. We deployed the fix two days before.
4. All the people was in the call.

<details>
<summary>Answers</summary>

1. There **are** many users online.
2. I went to **the** meeting yesterday.
3. We deployed the fix two days **ago**.
4. All the people **were** in the call.
</details>

---

**Tip**: When explaining technical concepts, always start with what the listener already knows, then build from there. Use everyday comparisons!
